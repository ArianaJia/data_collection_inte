## 2026-07-08
- 重构 publish 路径：除 CAN 发送外，其他任务只负责数据解析与更新缓存，由 task08 统一接收 publish 请求、进行 protobuf 编码，并通过 USART1_TX_DMA 发送；publish 请求目标已统一改为 task08 对应队列，不再指向 task01。
- 明确 CAN 发送分工：task04 的双请求中，热成像摘要仍可向 task08 发起 publish 请求，同时 CAN 摘要发送链路单独落实到 task09，对应 CAN1 发送队列保持独立。
- 推进 DMA 落地以避免大流量下阻塞：保留 task08 的 USART1_TX_DMA；task05 的 USART2 接收改为 ReceiveToIdle DMA + 环形缓冲；task06、task07 的 MCP2518FD SPI 收发切换为 DMA；task04 使用的 MLX90640/TCA9548A I2C1 关键收发路径也改为 DMA。
- 增加 task08 保护策略：启用 USART1 RTS/CTS 硬件流控，并加入 publish 主题去重与 10ms 冷却时间，优先保证系统稳定；其中 bms_detail 的业务语义不做削弱。
- 调整任务优先级并同步 .ioc：task08、task09 最高；task02、task03 次高；task04、task06 同一档；task05、task07 最低。
- 调整 MLX90640 初始化策略：取消 main.c 中可能阻塞启动的初始化逻辑，改为系统先进入 RTOS，再由 task01/defaultTask 记录状态并周期性执行健康巡检/恢复；task04 仅负责正常采样；调试日志继续通过 USART3 DMA 发送。
- 顺手修正了 MLX90640 I2C DMA 相关声明不一致的问题，并补充了离线状态记录与恢复日志，便于后续排查传感器掉线或初始化失败。

## 2026-07-14

### 调度架构重构
- 任务与队列全部切换为**静态分配**（StaticTask_t / StaticQueue_t），堆可用空间从 ~5KB 恢复至 ~15KB。
- 任务重命名，直接反映业务职责：
  - `myTask02` → `CanForViehcle`（整车 CANB 解码）
  - `myTask03` → `CanForBMS`（电池箱 CAN 解码）
  - `myTask04` → `MLX90640`（热成像采集）
  - `myTask05` → `TensionSensor`（RS485 拉力传感器）
  - `myTask06` → `SPICanControl`（SPI1/MCP2518FD CANA 轮询）
  - `myTask07` → `SPICanCDC`（SPI3/MCP2518FD CDC 监视）
  - `myTask08` → `Publish4G`（4G publish + USART1 发送）
  - `myTask09` → `CanBText`（CAN1 TX 发送）
- 新增 `initTaskBoot`（osPriorityLow1）：**集中完成所有模块初始化后自销毁**，初始化顺序为 MLX90640 → MCP2518FD-A → MCP2518FD-B → 4G AT 握手。各运行时任务不再自行初始化，仅检查就绪标志（`FDCAN_IsReady` / `MLX90640_App_IsSensorReady`）。
- 4G AT bootstrap **从 `MX_USART3_UART_Init` 前调度器阶段移除**，改为 `initTaskBoot` 中执行，消除启动时 15-20 秒阻塞。

### 函数命名规范化
- 所有内部辅助函数去掉 `Task0X_` 前缀，改为模块前缀：`CanVehicle_Dispatch`、`CanBattery_Dispatch`、`Mlx_SendSummary`、`Rs485_StartDma`、`SpiCana_Poll`、`SpiCdc_Poll` 等。

### 4G 模块通信优化
- USART1 AT 命令发送从 `HAL_UART_Transmit`（阻塞）改为 `HAL_UART_Transmit_DMA`（DMA + osDelay 轮询），不再忙等 CPU。
- `HAL_Delay` 全部替换为 `osDelay`，bootstrap 期间正常释放 CPU 给其他任务。

### Bug 修复
- AT bootstrap 全程零 HAL 阻塞：
  - 发送端：`HAL_UART_Transmit` → `Y100M_Uart1DmaSend`（USART1_TX_DMA + osDelay 轮询）
  - 接收端：逐字节 `HAL_UART_Receive` → `HAL_UARTEx_ReceiveToIdle_DMA`（USART1_RX_DMA + idle 回调）
  - 延时：`HAL_Delay` → `Y100M_DelayMs`（调度器运行后用 osDelay 让出 CPU）
  - `usart.c` 补 `#include "cmsis_os2.h"`（`osKernelRunning` 声明缺失导致编译报错）
- 查明确认 4G 模块 RTS/CTS 接线错误后，USART1 关闭硬件流控（`HwFlowCtl = DISABLE`）。

### 已知待修复
- SPI DMA 仅配置了 `hdmarx`，缺少 `hdmatx`，但 `mcp2518fd.c` 调用 `HAL_SPI_TransmitReceive_DMA` 要求两者均非空。当前无 `USE_FULL_ASSERT` 保护，解引用 NULL `hdmatx` 会导致 HardFault。需在 .ioc 中补充 SPI1_TX / SPI3_TX DMA 配置。
- USART2 中断已手动使能但 `stm32f4xx_it.c` 无 `USART2_IRQHandler`，idle 中断触发后落入 `Default_Handler` 死循环。需添加 handler 或改为纯 DMA 方式。
